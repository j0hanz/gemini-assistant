# Introduction

![Status: Planned](https://img.shields.io/badge/status-Planned-blue)

Restructure the four oversized tool modules in `src/tools/` (`analyze.ts` 622 LOC, `research.ts` 1180 LOC, `review.ts` 1226 LOC, `chat.ts` 1468 LOC) into focused folders with thin entrypoints plus pure local modules. Source plan: `.github/report.md`. All changes MUST preserve observable behavior (return values, thrown errors, side effects, structured output shape, progress/log text, retry counts, session resource URIs). Execution is wave-based; each wave gates on characterization tests plus `npm run lint && npm run type-check && npm run test` green.

## 1. Requirements & Constraints

- **REQ-001**: Zero observable behavior change across refactor (return shape, thrown errors, side effects, structured content fields, progress strings).
- **REQ-002**: Each PR corresponds to exactly one phase task row; one intent per commit.
- **REQ-003**: Wave 0 (characterization tests) MUST land before any production-code edit in waves 1–4.
- **REQ-004**: Every new module MUST export only what is consumed externally; internal helpers stay unexported.
- **REQ-005**: Public API of each tool registration function (`registerAnalyzeTool`, `registerResearchTool`, `registerReviewTool`, `registerChatTool`) MUST remain import-compatible from `src/tools/<name>.ts` OR via barrel `src/tools/<name>/index.ts` re-export; external imports MUST NOT break.
- **SEC-001**: No new code paths read env vars, spawn processes, or perform network I/O beyond existing surfaces.
- **CON-001**: Node `>=24`, TypeScript strict with `exactOptionalPropertyTypes: true`.
- **CON-002**: Retry count for JSON schema repair stays `JSON_REPAIR_MAX_RETRIES = 1`.
- **CON-003**: Deep research turn cap stays `MAX_DEEP_RESEARCH_TURNS = 6`.
- **CON-004**: Diff budget stays `MAX_DIFF_CHARS = 500_000`; risk scoring sets (`HIGH_RISK_SEGMENTS`, `LOW_SIGNAL_SEGMENTS`, `NOISY_EXACT_BASENAMES`, `NOISY_SUFFIXES`, `SOURCE_FILE_EXTENSIONS`, `ASSET_FILE_EXTENSIONS`) and weights stay byte-identical.
- **CON-005**: Session sanitization limits stay `SESSION_VALUE_MAX_STRING_LENGTH=2000`, `SESSION_VALUE_MAX_ARRAY_ITEMS=20`, `SESSION_VALUE_MAX_OBJECT_KEYS=50`.
- **GUD-001**: Follow refactor skill priority: name → extract → guard/type before classes/patterns.
- **GUD-002**: Rule of Three — extract shared helpers only on third duplication (Wave 5 gated by Waves 1–4).
- **PAT-001**: Folder layout per `.github/report.md` "Folder layout I would aim for" section.
- **PAT-002**: Each tool folder contains `index.ts` (registration + thin entry), plus topic-named siblings; no deep nesting beyond `research-phases/` and `review-diff/`.

## 3. Alternatives

- **ALT-001**: Single big-bang rewrite into new folder layout in one PR. Rejected: unbisectable, impossible to review, violates refactor skill "one intent per commit" and risk-tier escalation rules.
- **ALT-002**: Start with `chat.ts` (highest-ROI-by-LOC). Rejected: it carries retry + session + persistence tripwires; refactor skill mandates "leave hardest for last" once a repo pattern exists.
- **ALT-003**: Immediately extract a shared `model-stream-runner` across all four tools (Wave 5 first). Rejected: premature abstraction; violates Rule of Three until Waves 1–4 expose the true shared shape.
- **ALT-004**: Sub-split deep research into five phase files (`plan/retrieve/synthesize/contradiction/aggregate`) per report PR 2.2 exact spec. Rejected for Phase 2; deferred to follow-up because single-function extraction first avoids premature module proliferation.
- **ALT-005**: Convert tool entrypoints to classes. Rejected: refactor skill prefers module-level functions; no state or inheritance justifies classes.

## 7. Risks & Assumptions

- **RISK-001**: Test files import unexported helpers from current single files; folder move could break internal imports. Mitigation: re-export all test-consumed symbols from `index.ts` barrel.
- **RISK-002**: Circular imports when splitting `chat.ts` between `chat-session.ts` (depends on output shape) and `chat-output.ts`. Mitigation: keep shared types in a small `chat-types.ts` if needed; detect via `npx madge --circular --extensions ts src/`.
- **RISK-003**: `exactOptionalPropertyTypes` regressions when passing `structured.foo` through new helper signatures. Mitigation: match current conditional-spread pattern; no widening.
- **RISK-004**: Deep research phase sub-split creates many tiny modules with no cohesion (report PR 2.2 over-slice risk). Mitigation: defer sub-split, extract as single `research-deep.ts` first.
- **RISK-005**: Snapshot drift from unrelated LLM output changes. Mitigation: tests mock `executor.runStream` and `generateContentStream` — no live model calls.
- **RISK-006**: Progress-message text changes would silently break downstream log consumers. Mitigation: Phase 0 TASK-004 asserts exact strings.
- **ASSUMPTION-001**: No external consumer imports from deep paths inside `src/tools/*.ts` beyond what tests and `src/index.ts` / `src/server.ts` already import.
- **ASSUMPTION-002**: The existing 31-test suite already exercises all tool registration paths, so registration regressions surface immediately.
- **ASSUMPTION-003**: Existing helpers (`pickDefined`, `buildGenerateContentConfig`, `executor.runStream`, `ProgressReporter`) remain stable across the refactor window.
- **ASSUMPTION-004**: No feature work lands on `src/tools/*` during refactor; if it does, waves rebase cleanly because splits are mechanical.
